## 2025-05-15 - [Accessibility & Modal UX Enhancements]
**Learning:** Monolithic single-file applications require consistent accessibility coverage across both static HTML and JavaScript-driven UI updates. Updating `textContent` on buttons with complex child structures (like notification badges) can lead to UI state loss and runtime errors. Additionally, programmatic focus management is essential for large-scale modal-heavy interfaces to ensure keyboard accessibility and avoid "focus trap" issues.
**Action:** Always wrap button icons in dedicated span elements when child elements (like badges) exist to prevent overwriting during state transitions. Implement centralized focus management in modal utility functions using a slight delay (e.g., 300ms) to accommodate CSS opening transitions. Ensure all icon-only buttons in both static and dynamic contexts have descriptive `aria-label` and `title` attributes.
